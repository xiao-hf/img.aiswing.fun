# 宝塔/Nginx 最短操作步骤

## A. 覆盖静态文件

把 `delivery/img-site-upload.zip` 解压到网站根目录：

`/www/wwwroot/img.aiswing.fun/`

覆盖同名文件。

必须能访问：

`https://img.aiswing.fun/version.txt`

看到：

`img.aiswing.fun static build 2026050603`

如果还是 404，说明上传目录错了或没覆盖成功。

## B. 删除错误文件

服务器根目录里如果有下面文件，删掉：

- `api.php`
- `server.js`
- `package.json`
- `server.out.log`
- `server.err.log`

`api.php` 在你当前服务器会直接输出源码，不能留。

## C. 配置 Nginx 反代

宝塔：网站 -> img.aiswing.fun -> 配置文件。

在 `server { ... }` 里面加入：

```nginx
location ^~ /v1/images/ {
    proxy_pass https://gpt.aiswing.fun;
    proxy_ssl_server_name on;
    proxy_set_header Host gpt.aiswing.fun;
    proxy_set_header X-Real-IP $remote_addr;
    proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    proxy_set_header X-Forwarded-Proto $scheme;

    proxy_connect_timeout 60s;
    proxy_send_timeout 300s;
    proxy_read_timeout 300s;
    client_max_body_size 50m;
    proxy_request_buffering off;
}
```

保存后点“重载配置”或执行：

```bash
nginx -t && nginx -s reload
```

## D. Cloudflare 清缓存

清缓存：

- `https://img.aiswing.fun/`
- `https://img.aiswing.fun/app.js`
- `https://img.aiswing.fun/index.html`
- `https://img.aiswing.fun/version.txt`

## E. 诊断

访问：

`https://img.aiswing.fun/diagnose.html`

三个检查都应该通过：

1. `version.txt 覆盖检查`
2. `app.js 版本检查`
3. `/v1/images 反代检查`

其中第 3 个用无效 key 测试时，可能返回 401 JSON，这是正常的；只要不是 `404 nginx` 或 `Failed to fetch`，就说明反代打通。
